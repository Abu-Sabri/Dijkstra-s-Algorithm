What is Dijkstra's Algorithm?

Dijkstra's algorithm, conceived by Dutch computer scientist Edsger W. Dijkstra in 1956, finds the shortest path from a source vertex to every other vertex in a graph with non-negative edge weights. It is one of the most fundamental algorithms in graph theory and forms the backbone of routing protocols, navigation systems, and network optimization.

For our Bahrain network, the weight is the road distance in kilometers. Once the algorithm finishes, we can read the shortest distance from Manama to any other city, including Durrat Al Bahrain.

Step-by-Step Walkthrough

1
Initialization
Set the distance to the source vertex s to 0 and the distance to every other vertex to ∞ (infinity). Mark all vertices as unvisited. Create a priority queue seeded with the source.
2
Selection
Repeatedly extract the unvisited vertex with the smallest known distance from the priority queue. This is the current vertex u. On the first iteration it is always the source.
3
Relaxation
For every unvisited neighbor v of u, compute a candidate distance alt = dist[u] + w(u, v). If alt < dist[v], update dist[v] = alt and record prev[v] = u. Push the new distance into the priority queue.
4
Marking
Mark vertex u as visited. Once visited, its distance is final — it can never be improved. (This invariant is what makes Dijkstra's algorithm correct.)
5
Repetition
Loop back to step 2. Stop when the destination is visited (or when the priority queue is empty — in which case the destination is unreachable).

Interactive Walkthrough — Bahrain Example

Let's trace through the first few iterations manually so you know what to look for in the visualizer:

  1. Init — dist[Manama] = 0, all other distances = ∞.
  2. Visit Manama (dist 0). Relax its neighbors:
    • Muharraq: 0 + 6 = 6
    • Isa Town: 0 + 10 = 10
    • Sitra: 0 + 12 = 12
    • Budaiya: 0 + 14 = 14
  3. Pick smallest unvisited → Muharraq (6). But Muharraq has no unvisited neighbors that connect to anything new, so it's marked visited with distance 6. (Dead end relative to our destination.)
  4. Pick next smallest → Isa Town (10). Relax its neighbors:
    • Rifa: 10 + 6 = 16
    • Hamad Town: 10 + 8 = 18
    • Sitra: 10 + 9 = 19 — but Sitra is already 12, so no update! Direct Manama→Sitra wins.
  5. Pick next → Sitra (12). Relax: Durrat = 12 + 12 = 24 ✅
  6. Pick next → Budaiya (14), then Hamad Town (18), then Rifa (16, also leads to Durrat at 34 — worse), etc.

The algorithm terminates with dist[Durrat] = 24 and the path reconstructed via prev[Durrat] = Sitra, prev[Sitra] = Manama → Manama → Sitra → Durrat Al Bahrain.

Why is Manama → Sitra → Durrat (24) better than Manama → Isa Town → Rifa → Durrat (10+6+18 = 34)? Because at every step Dijkstra expands the cheapest frontier, the direct Manama→Sitra edge stays as dist[Sitra] = 12. By the time we consider Rifa, even its 6 km to Isa Town cannot beat the already-known 12.

Key Concepts

Weighted Graph
A graph where each edge carries a numerical cost — here, road distance in kilometers.
Priority Queue
A data structure (binary heap, Fibonacci heap, …) that always returns the smallest element in O(log n) — crucial for efficiency.
Greedy Approach
Once a node is marked visited, its shortest distance is final. This greedy choice is provably correct when all weights are non-negative.
Relaxation
Updating a neighbor's tentative distance when we find a shorter route through the current node. The heart of the algorithm.

Common Mistakes

❌ Re-visiting a visited node. A visited node's distance is already final. Re-processing it wastes time and can introduce subtle bugs if you mutate dist after the fact.
❌ Forgetting to relax all unvisited neighbors. Skipping one means the shortest path to that neighbor is never found.
❌ Forgetting to update prev. Without it, you can compute the distance but cannot reconstruct the path.
✅ Use a priority queue. A linear scan of all unvisited nodes makes the algorithm O(V²). A binary heap brings it down to O((V+E) log V).

Formal Definition

A weighted graph is a triple G = (V, E, w) where:

  • V is a finite set of vertices (cities): { Manama, Muharraq, Isa Town, Sitra, Budaiya, Rifa, Hamad Town, Durrat Al Bahrain }
  • E ⊆ V × V is a set of undirected edges (roads)
  • w: E → ℝ≥0 is a weight function assigning a non-negative real number to each edge (distance in km)

Input. A weighted graph G, a source vertex s ∈ V, a target vertex t ∈ V.

Output. The shortest-path distance d(s, t) and a shortest path P from s to t.

Algorithm Pseudocode

function Dijkstra(G, s, t):
    // dist[v]  = best known distance from s to v
    // prev[v]  = predecessor of v on the best known path
    // visited  = set of finalized vertices

    for each v in V:
        dist[v]  = ∞
        prev[v]  = null
        visited[v] = false

    dist[s] = 0
    Q = priority_queue()
    Q.push((0, s))

    while Q is not empty:
        (d, u) = Q.pop()        // smallest distance first
        if visited[u]: continue
        visited[u] = true
        if u == t: break      // reached destination

        for each (u, v, w) in G.edges_from(u):
            if not visited[v]:
                alt = dist[u] + w      // candidate distance
                if alt < dist[v]:
                    dist[v] = alt
                    prev[v] = u
                    Q.push((alt, v))

    return dist[t], reconstruct_path(prev, s, t)

Correctness Sketch

We prove by induction on the set of visited vertices. Let Vk be the first k vertices added to visited. The induction hypothesis is that for every v ∈ Vk, dist[v] equals the true shortest distance from s to v.

Base. V₁ = {s} and dist[s] = 0, which is trivially the shortest distance from s to itself.

Step. Suppose the hypothesis holds. The next vertex u selected is the unvisited vertex with minimum dist[u]. Any alternative path to u must go through an unvisited vertex with distance ≥ dist[u]. Since all edge weights are non-negative, that path is at least dist[u] long. Hence dist[u] is already optimal. ∎

Why non-negative weights? Dijkstra's correctness depends on the invariant that once we mark a vertex visited, no later path through unvisited vertices can be shorter. If negative weights existed, a later path could decrease a previously final distance — and we would need Bellman-Ford instead.

Complexity Analysis

Data structureTime complexityNotes
Array / linear scanO(V²)Simple, fine for dense small graphs
Binary heap (min-heap)O((V + E) log V)Default choice — used in this simulator
Fibonacci heapO(E + V log V)Theoretical optimum, rarely used in practice

For the Bahrain network: V = 8, E = 10, so the algorithm visits at most 8 nodes and relaxes at most 10 edges — essentially instantaneous.

Variants & Extensions

  • A* search — adds a heuristic to guide the search toward the target.
  • Bidirectional Dijkstra — runs the search forward from s and backward from t simultaneously, often 2× faster in road networks.
  • Bellman-Ford — handles negative weights, but at O(VE) cost.
  • Floyd-Warshall — finds all-pairs shortest paths in O(V³).
